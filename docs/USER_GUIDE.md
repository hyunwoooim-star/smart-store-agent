## 1. 이게 뭐하는 프로그램이야?

### 한 줄 요약
**1688에서 상품 보고 "이거 팔면 남을까?" 궁금할 때, 5초 만에 답 알려주는 도구**

### 문제 상황
```
"1688에서 캠핑의자 45위안(약 9,000원)에 팔길래
네이버에서 45,000원에 팔면 36,000원 남겠네? 대박!"

→ 실제로 해보면: 적자 😭
```

### 왜 적자가 나나요?

| 숨겨진 비용 | 금액 |
|------------|------|
| 상품원가 | 8,775원 (45위안 × 195원) |
| 중국 내 배송비 | 3,000원 |
| 구매대행 수수료 (10%) | 1,178원 |
| 해외 배송비 (항공) | 32,000원 (4kg × 8,000원) |
| 관부가세 (20%) | 8,791원 |
| 국내 택배비 | 3,500원 |
| 네이버 수수료 (5.5%) | 2,475원 |
| 반품/CS 충당금 (5%) | 2,250원 |
| 광고비 (10%) | 4,500원 |
| 포장비 | 500원 |
| **총 비용** | **66,969원** |
| **판매가** | 45,000원 |
| **손익** | **-21,969원 (적자!)** |

### 이 프로그램이 해주는 일
1. **모든 숨겨진 비용 자동 계산** - 빠뜨리는 거 없음
2. **부피무게 vs 실무게 비교** - 배송비 바가지 방지
3. **위험도 신호등** - 🔴🟡🟢 한눈에 판단
4. **손익분기 판매가 알려줌** - "최소 ○○원에 팔아야 남는다"
5. **네이버 대량등록 엑셀 자동 생성** - 복붙 노가다 끝

## 2. 시작하기 전에 알아야 할 것들

### 핵심 용어 정리

| 용어 | 뜻 | 예시 |
|------|-----|------|
| **CNY** | 중국 위안화 | 45 CNY = 약 8,775원 |
| **부피무게** | 크기로 계산한 무게 | (가로×세로×높이)/6000 |
| **청구무게** | 실제 배송비 기준 | max(실무게, 부피무게) |
| **MOQ** | 최소 주문 수량 | MOQ 50개 = 최소 50개 사야 함 |
| **마진율** | 순이익 ÷ 판매가 × 100 | 30% 이상이 안전 |
| **손익분기가** | 이 가격 이상 팔아야 본전 | 변동비 고려한 최저가 |

### 환경 설정 기본값

```python
환율 = 195원/위안           # 2026년 1월 기준
항공 배송비 = 8,000원/kg    # 배대지 평균
해운 배송비 = 75,000원/CBM  # 대량 주문용
국내 택배 = 3,500원/건
네이버 수수료 = 5.5%
반품 충당금 = 5%            # 필수! 빠뜨리면 망함
광고비 = 10%               # 초반 노출용
```

## 3. Step-by-Step 사용법

### Step 1: 프로그램 실행

```bash
# Windows (WSL)
cd ~/smart
source .venv/bin/activate
streamlit run src/ui/app.py

# 브라우저에서 열기
# http://[WSL IP]:8501
```

### Step 2: 상품 정보 입력

1. **상품명**: 네이버에 등록할 이름 (100자 이내)
2. **1688 가격 (CNY)**: 중국 도매가 (위안 단위)
3. **무게 (kg)**: 실제 무게 (1688 상세에서 확인)
4. **크기 (cm)**: 가로 × 세로 × 높이 (포장 기준!)
5. **MOQ**: 최소 주문 수량
6. **목표 판매가**: 네이버에서 팔 예정 가격

### Step 3: 리스크 분석 버튼 클릭

결과 화면 해석:

```
┌─────────────────────────────────────────┐
│ 📊 마진 분석 결과                        │
├─────────────────────────────────────────┤
│ 🔴 위험 등급: DANGER                     │
│                                         │
│ 총 비용: 66,969원                        │
│ 판매가: 45,000원                         │
│ 예상 손익: -21,969원                     │
│ 마진율: -48.8%                          │
│                                         │
│ 💡 손익분기 판매가: 85,000원             │
│ 💎 30% 마진 달성가: 122,000원            │
│                                         │
│ 추천: 🔴 진입 금지! 최소 85,000원 이상 필요│
└─────────────────────────────────────────┘
```

### Step 4: 결과 해석 및 의사결정

| 결과 | 행동 |
|------|------|
| 🟢 **SAFE** (마진 30%↑) | 바로 시작! 소량 사입 진행 |
| 🟡 **WARNING** (마진 15~30%) | 샘플 테스트 후 결정 |
| 🔴 **DANGER** (마진 15%↓) | 손대지 마! 다른 상품 찾기 |

### Step 5: 엑셀 다운로드 (대량 등록용)

1. "목록에 추가" 버튼 클릭
2. 여러 상품 분석 후 추가
3. "엑셀 다운로드" 버튼 클릭
4. `naver_products.xlsx` 파일 저장
5. 네이버 판매자센터 → 대량등록 → 파일 업로드

## 4. 마진 계산 공식 완전 해부

### 전체 비용 구조

```
총비용 = 고정비용 + 변동비용

고정비용:
├── 상품원가 (CNY × 환율)
├── 중국 내 배송비 (3,000원)
├── 구매대행 수수료 (원가의 10%)
├── 관부가세 (과세가격의 약 20%)
├── 해외 배송비 (청구무게 × kg당 요금)
├── 국내 택배비 (3,500원)
└── 포장비 (500원)

변동비용 (판매가 기준):
├── 마켓 수수료 (5.5% ~ 15%)
├── 반품 충당금 (5%)
└── 광고비 (10%)
```

### 부피무게 계산 (핵심!)

```python
부피무게 = (가로 × 세로 × 높이) / 6000

# 예시: 캠핑의자
실제무게 = 2.5kg
부피무게 = (80 × 20 × 15) / 6000 = 4.0kg
청구무게 = max(2.5, 4.0) = 4.0kg  # 더 큰 쪽!

# 배송비
항공: 4.0kg × 8,000원 = 32,000원
```

**주의**: 큰 물건은 무게보다 크기로 돈 나간다!

### 손익분기 판매가 공식

```python
변동비율 = 마켓수수료 + 반품충당금 + 광고비
        = 5.5% + 5% + 10%
        = 20.5%

손익분기가 = 고정비용 ÷ (1 - 변동비율)
         = 고정비용 ÷ 0.795
```

### 30% 마진 달성가 공식

```python
목표마진가 = 고정비용 ÷ (1 - 변동비율 - 목표마진)
          = 고정비용 ÷ (1 - 0.205 - 0.30)
          = 고정비용 ÷ 0.495
```

## 5. 위험도 판정 기준

### 신호등 체계

| 등급 | 마진율 | 의미 | 행동 지침 |
|------|--------|------|-----------|
| 🟢 **SAFE** | 30% 이상 | 안전 | 적극 진입 |
| 🟡 **WARNING** | 15~30% | 주의 | 샘플 테스트 |
| 🔴 **DANGER** | 15% 미만 | 위험 | 진입 금지 |

### 왜 15% 미만이 위험한가?

```
마진율 15% 상황:

판매가: 100,000원
마진: 15,000원

예상치 못한 비용:
- 반품 1건: -100,000원 (원복불가)
- 불량 클레임: -50,000원
- 환율 5% 상승: -5,000원

→ 한두 번 일 터지면 바로 적자전환
```

### MOQ 고려 의사결정

| MOQ | 마진 30%+ | 마진 15~30% | 마진 15%↓ |
|-----|-----------|-------------|-----------|
| 1~5개 | 🟢 바로 시작 | 🟡 테스트 진행 | 🔴 포기 |
| 6~20개 | 🟢 샘플 후 시작 | 🟡 협상 시도 | 🔴 포기 |
| 21~50개 | 🟡 품질 먼저 확인 | 🟡 협상 필수 | 🔴 포기 |
| 50개+ | 🟡 구매대행 테스트 먼저 | 🔴 포기 | 🔴 포기 |

## 6. 전략 가이드: 이렇게 써야 돈 번다

### 전략 1: "망하는 상품 필터링" (가장 중요!)

```
❌ 나쁜 습관:
"이거 좋아보이네 → 바로 주문"

✅ 올바른 순서:
1. 이 프로그램으로 마진 계산
2. 🔴이면 무조건 패스
3. 🟡이면 샘플만 주문
4. 🟢이면 소량 사입 시작
```

### 전략 2: 부피무게 함정 피하기

**위험 상품 (부피무게 >> 실무게)**:
- 캠핑 텐트, 의자
- 대형 수납함
- 부피 큰 인형
- 빈 용기류

**안전 상품 (실무게 ≈ 부피무게)**:
- 소형 전자기기
- 악세서리, 공구
- 화장품, 식품
- 의류 (압축 가능)

```python
# 경험 법칙
if 부피무게 > 실무게 × 2:
    print("⚠️ 배송비 폭탄 주의!")
```

### 전략 3: 마켓별 진입 전략

| 마켓 | 수수료 | 특징 | 추천 상품 |
|------|--------|------|-----------|
| 네이버 | 5.5% | 신규 셀러 친화적 | 틈새시장 |
| 쿠팡 | 10.8% | 물류 자동화 | 생활용품 |
| 아마존 | 15% | 글로벌 | 고마진 필수 |

**팁**: 초보는 네이버로 시작 → 검증된 상품만 쿠팡 확장

### 전략 4: 시즌 상품 주의

```
❌ 실수:
"여름에 선풍기 대박이네! 100개 주문!"
→ 8월에 도착 → 시즌 끝 → 재고 폭탄

✅ 올바른 접근:
1. 시즌 2개월 전 소량 테스트 (5~10개)
2. 반응 보고 리오더
3. 시즌 끝 1개월 전 발주 중단
```

### 전략 5: 손익분기가 활용법

```python
# 시나리오 분석
손익분기가 = 85,000원
경쟁사 최저가 = 75,000원

if 손익분기가 > 경쟁사최저가:
    print("❌ 가격 경쟁력 없음! 다른 상품 찾기")
else:
    print("✅ 경쟁 가능! 진입 고려")
```

## 7. 자주 묻는 질문

### Q1: 환율이 바뀌면 어떻게 해요?

설정에서 변경 가능:
```python
# src/core/config.py
exchange_rate = 200  # 195 → 200원으로 변경
```

### Q2: 배대지 요금이 올랐어요

```python
# src/core/config.py
shipping_rate_air = 9000  # 8000 → 9000원으로 변경
```

### Q3: 쿠팡으로 팔고 싶은데요?

분석 화면에서 "판매 마켓" 선택:
- 네이버: 수수료 5.5%
- 쿠팡: 수수료 10.8%
- 아마존: 수수료 15%

### Q4: 반품 충당금 5%가 너무 높은 것 같아요

**절대 빼지 마세요!** 실제 데이터:
- 의류: 반품률 15~25%
- 전자기기: 반품률 3~5%
- 평균: 5~10%

5%는 "최소한"의 안전마진입니다.

### Q5: 광고비 안 쓰면 10% 빼도 되나요?

**비추천!** 이유:
- 초반 노출 없으면 판매 0건
- 최소 3개월은 광고 필요
- 광고비 0원 = 매출 0원

### Q6: 1688 스크래핑이 안 돼요

현재 Mock 데이터 사용 중. 실제 연동은 Phase 7에서 Apify API로 구현 예정.
지금은 수동으로 1688에서 정보 확인 후 입력하세요.
